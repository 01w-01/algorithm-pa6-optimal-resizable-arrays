- 本论文讲的是更省空间的可变数组
- 传统的可变数组使用倍增法，平摊分析时间为O（1）
- 但是要额外占用O（N）的空间
- 例如：如果存（1024+1）个元素，存入1025个时将1024个进行倍增
- 2048个空间只存入了1025个
- 所以会额外占用近N个空位
- 更优的是空间为 N + O(√N) ，时间平摊分析依旧为O（1）
- 本篇论文提出把空间分为数组占用和扩容时临时借用的两种
- 对于任意整数 r ≥ 2，本文证明：
- 在增长和收缩操作期间可以短暂使用 N + O(N^{1−1/r}) 空间
- 那么存储并访问一个大小为 N 的数组只需要 N + O(N^{1/r}) 空间
- 按下标访问需要 O(1) 最坏情况时间
- 增长和收缩操作需要 O(r) 平摊时间
- 即空间变小了，不过代价是增长和收缩时间变久
