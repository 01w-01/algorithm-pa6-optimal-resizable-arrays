## Optimal resizable arrays
> 最优可伸缩数组  
> optimal：最优  
> resizable arrays：可伸缩数组、动态数组

## 论文核心
做一个数组：

- 能够 $O(1)$ 访问
- 能够动态增长
- 把空着但已占据的内存压到极小

## 背景
论文解决的问题的重要性:  
不管是c语言,c++,还是python,底层都使用到了数组这一数据结构,所以减少资源消耗很重要
> 目前没有看到直接推翻或者大幅推进该论文的后续工作 ,所以,作者所说的,基本把这个问题解决完了站的住

历史背景与发展(对应section 3之前):
1. 使用倍增(doubling)的方法
    - 使用$2$倍的临时空间,然后直接复制过去
	   - 空间
	       - $O(N)$
	   - 时间
	       - $O(1)$均摊
2. 每次申请大小恰好为$N$的数组
    - 空间
       - $N+O(1)$
    - 时间
       - $O(N)$
3. 使用Sitarski / Brodnik 的 $N+O(\sqrt{N})$
	- 把大数组切为多个小块,每个小块$\sqrt{N}$
		 - 空间
	       - 索引(index block)块:$\sqrt{N}$+最后一个没填满的块
    - 作用:
       - 平衡了时间和空间的消耗
4. 最后:也就是本论文,学习了Sitarski / Brodnik 的 $N+O(\sqrt{N})$
    使之更具有普遍性


## section 1 问题和重要性
### resizable array 解决的抽象问题
普通数组是定长的,申请数组时就定好了大小N,不能修改
从而引发抽象问题
> 是否有一种数据结构
> 既能想普通数组一样 get(i)/set(i，a)一样常数时间访问,又能想列表一样在端侧增加或者删除元素,即grow,shrink,并且使用的空间消耗不大

->resizable array的发展
### 为什么 contiguous array(连续数组) 的 grow 难？
> 因为有O(1)随机访问的前提在,所以内存必须是一整块连续的

此对应了每次数组达到上限,就grow 1,空间消耗达到极限,但是时间复杂度特别大
### doubling 的浪费在哪里？
doubling即倍增法
是在空间达到上限时,申请了(1+a)倍的空间,从原有的复制过去,空间换时间
shrink类似,当元素少到空间上限的1/(1+a)^2时
### $N+O(\sqrt N)$ 旧结果为什么强？
> 结论:
> 做到了使用空间少的同时还访问快
> 即做到了空间和时间复杂度的均衡,减少

做法:
将数组分为大小为/sqrt{N}的块(block),然后再使用索引块(index block)
浪费在于index block和最后一个没有填满的数据块,都是/sqrt{N}的空间复杂度,远小于doubling的O(N)
### 本文为什么能“突破”旧下界？
Brodnik 证明了 N+Ω(√N) 是下界,这是在一个隐形前提下确定的
> 稳态空间和resize临时空间看做一个量

也就是说本文其实就是把稳态空间和resize临时空间拆开,在更细粒度的层面进行讨论,从而做到优化

### Section 2：模型定义
> 定义了本文依赖的假设

ADT：抽象数据类型

- `Get(i)`：读取第 `i` 个元素
- `Set(i, a)`：把第 `i` 个元素改成 `a`
- `Grow(a)`：在数组末尾新增一个元素 `a`
- `Shrink()`：删除数组末尾的元素

Memory model：内存模型

- `fixed-length allocated blocks`：已经申请好的固定长度内存块
- `data blocks contain items`：数据块存真正的数组元素
- `index blocks contain pointers`：索引块存指针，用来找到数据块
- `space = sum of lengths of allocated blocks`：总空间 = 所有已申请内存块长度之和

Definition：定义

- `(s(N), t(N))-implementation`：一种实现方式，用两个函数描述它的额外空间开销
- `storage state uses <= N + s(N)`：平时存一个长度为 `N` 的数组，最多用 `N + s(N)` 空间
- `during grow/shrink uses <= N + t(N)`：执行扩容或缩容时，临时最多用 `N + t(N)` 空间

这里的核心区别：

- `s(N)`：平时多占多少空间
- `t(N)`：调整大小时临时多占多少空间
- 论文的关键就是把这两个空间分开分析
