问得好，这正是 Section 5/6 里容易被一带而过、但其实是保证"均摊 O(1)/O(r)"和"重建瞬间空间不爆�ns"这两件事同时成立的关键机制。我按 r=3（Section 5 的结构）来讲，逻辑和 Section 6 的一般 r 是一致的。

## 1. 什么时候触发重建

回忆参数约束：`N^{1/3} ≤ B ≤ 4N^{1/3}`，等价地写成 `(B/4)³ ≤ N ≤ B³`。

- **Grow 方向**：当 `N` 涨到 `B³` 时（此时 large block 已经有 `N/B²=B` 个，全部占满，small block 那边也再挤不出空间了），把 `B` **加倍**，整个结构按新的 `B` 重建。
- **Shrink 方向**：对称地，当 `N` 缩到 `(B/4)³` 时，把 `B` **减半**，重建。

也就是说 `B` 不是一直不变的，它每隔一段"生长期"就翻倍一次（或减半一次），这样才能一直维持 `N^{1/3}≤B≤4N^{1/3}` 这个不变量。

## 2. 重建瞬间空间复杂度是多少

**答案：还是 `O(N^{2/3})`，不会更多**——这正是它必须保持在 `t(N)` 那个量级、否则就白设计了。

诀窍在于：**不是先把新结构整个建好、旧结构还留着，最后一起释放**（那样瞬间要 `2N` 空间），而是像 Sitarski 的 HAT 重建一样，**逐块搬**：

```
旧结构（B）                     新结构（2B）
[B²][B²][B²]...            →   [(2B)²]   ← 正在被填的新 large block
[B²][B²][B²]...                （从旧结构里按顺序读元素填进来）
```

具体过程：
1. 分配**一个**新的大小 `(2B)²` 的 large block；
2. 按顺序（`a_0, a_1, ...`）把旧结构里的元素一个个/一批批拷贝进这个新 block；
3. 旧的那些小 block 一旦被拷完，**立刻释放**，不用等全部搬完；
4. 新 block 填满了，再分配下一个 `(2B)²` 的 block，继续搬。

因为任何时刻，**同时存在的"额外"内存**只是"正在被填的那一个新 large block（大小 `(2B)²=O(N^{2/3})`）"，旧结构里没搬到的部分本来就占着空间（属于 N 本身），已经搬完的部分已经释放了——所以任何时刻总空间都还是 `N + O(N^{2/3})`，没有超过 `t(N)` 的量级。

这跟 HAT 那句话（Section 3.2）是同一个原理：

> "a rebuild operation can be carried out while using only O(√N) extra storage. The new and old data blocks are allocated and deallocated one by one."

这里只是把 `√N` 换成了 `N^{2/3}`。

## 3. 均摊代价怎么摊掉

重建这一次性搬 `N` 个元素的代价是 `O(N)`，看起来很大，但可以摊到"上一次重建到这一次重建之间发生的操作数"上：

- 上一次重建把容量设定为对应 `B/2` → `N` 从 `(B/2)³ = N/8` 开始；
- 这一次重建发生在 `N=B³`；
- 所以中间至少发生了 `N - N/8 = (7/8)N = Ω(N)` 次 Grow 操作。

用这 `Ω(N)` 次操作去分摊 `O(N)` 的重建代价，每次操作均摊下来只多付 `O(1)`——这跟"倍增法 vector 扩容均摊 O(1)"的经典论证是一模一样的算术，只是这里"翻倍"的对象是 `B` 而不是数组本身。

Shrink 方向同理：`(B/4)³` 到 `(B/2/4)³` 之间也至少经过 `Ω(N)` 次 Shrink 才会再次触发重建。

## 4. 小结一下这三件事的关系

| 量 | 大小 | 保证了什么 |
|---|---|---|
| 稳态空间 `s(N)` | `O(N^{1/3})` | 平时（不重建时）浪费多少 |
| 临时空间 `t(N)` | `O(N^{2/3})` | 重建/合并瞬间最多多用多少（逐块搬保证不超这个数） |
| 重建频率 | 每 `Ω(N)` 次操作一次 | 保证 `O(N)` 的重建代价均摊到每次操作只有 `O(1)` |

这三条凑在一起，才让"稳态 `N^{1/3}`、均摊 `O(1)`"这个承诺真正成立，而不是纸面上好看、实际重建时炸内存。
