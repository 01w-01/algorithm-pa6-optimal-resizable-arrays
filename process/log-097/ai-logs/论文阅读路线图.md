## 2. 精读 vs 泛读

这是 review 的问题定义和贡献来源。重点抓四句话：

### Section 4：下界证明

这是你 review 的第一个证明核心。笔记要按“可验证链条”写。

```md
Theorem 4.2:
Any (s(N), t(N))-implementation must satisfy s(N)t(N) >= N.

Proof skeleton:
1. After N grow operations, N items are stored in some number k of blocks.
2. Each block needs a pointer, so k <= s(N).
3. Therefore some block has size at least N / s(N).
4. When this block was allocated, it was temporarily extra space.
5. Therefore t(N) >= N / s(N).
6. Hence s(N)t(N) >= N.
```

review 中用这个定理解释最优性：

* 若稳定额外空间 $s(N)=O(N^{1/r})$；
* 则临时额外空间必须 $t(N)=\Omega(N^{1-1/r})$；
* 本文构造达到 $O(N^{1-1/r})$；
* 所以临时空间 trade-off 是 tight 的。

Parameters:

`Combine-Blocks()` 笔记：

```md
When level 1 is full:
- find smallest k with nk < 2B
- for i = k-1 down to 1:
    combine B blocks of size B^i
    into one block of size B^{i+1}
- this resembles carrying in a redundant base-B counter
```

`Split-Blocks()` 笔记：

```md
When shrink needs an item but no B-block exists:
- find smallest k with nk > 0
- split one B^k block down into smaller blocks
- produces blocks at lower levels so future shrinks are cheap
```

### Section 6.2：摊还分析

你没学过摊还分析，建议用“信用卡记账法”理解。

Grow 的信用规则：

```md
Each item at level i has r - i credits.
New item enters level 1, so it receives r - 1 credits.
When item moves from level i to i+1, its credit decreases by 1.
That released credit pays for the copy.
Therefore all Combine-Blocks copying is prepaid.
Grow amortized cost = O(r).
```

Shrink 的信用规则：

```md
Each Shrink adds O(1) credits to each level.
After enough shrink operations, a split at level k has accumulated enough credits.
These credits pay for:
1. copying the B^k items,
2. assigning extra credits to items moved to lower levels.
Therefore Split-Blocks amortized cost = O(r).
```

Rebuild 的信用规则：

```md
Rebuild happens only when N changes by a constant factor:
- grow side: B doubles
- shrink side: B halves

Many operations occur between rebuilds.
Their accumulated credits pay for rebuilding N items.
```

review 里不用复现每个不等式，但要保留关键量级：

* Combine cost creating $B^k$ block is $O(B^k)$；
* Split cost at level $k$ is $O(B^k)$；
* Rebuild cost is $O(N)$；
* 每个 operation 多付 $O(r)$ credit，足够覆盖未来搬移。

Optimized access:

Game:

## 5. 写 review 时“正确性和优越性”需要验证哪些细节

### 正确性验证

你要让同学能检查数据结构是否真的表示了同一个 array。

记录以下证明点：

1. **顺序不变性**
   blocks 虽然分层，但 array items 的逻辑顺序保持一致。Combine 是把连续的低层 blocks 按顺序拷贝到高层 block；Split 是把高层 block 按顺序拆回低层 blocks。

2. **容量不变性**
   每层最多 $2B$ 个 block，level 1 允许一个 partially filled block。这样 grow 总有规则处理，shrink 也能通过 split 找到低层 block。

3. **访问正确性**
   给定 index $i$，能够确定它落在哪个 level、哪个 block、block 内偏移。朴素版本 $O(r)$ 查层已经能证明正确，bit-trick 只负责把查层加速到 $O(1)$。

4. **Grow 正确性**
   新 item 总是成为逻辑末尾元素。若低层满，则先 Combine 腾出结构空间，再插入。

5. **Shrink 正确性**
   删除的总是逻辑末尾元素。若没有低层 block，则 Split 一个更高层 block，把末尾附近元素拆出来再删。

### 空间优越性验证

你要记录三个空间量：

```md
Stable storage:
- data items: exactly N
- index/metadata: O(rB)
- with B ≈ N^{1/r}
- Section 6 gives N + O(rN^{1/r})
- Section 7 improves to N + O(N^{1/r})

Temporary resize:
- largest newly allocated block size: B^{r-1}
- B^{r-1} ≈ N^{1-1/r}
- temporary space = N + O(N^{1-1/r})
```

再连接下界：

```md
Theorem 4.2:
s(N)t(N) >= N

If s(N)=O(N^{1/r}),
then t(N)=Omega(N^{1-1/r}).

The construction has t(N)=O(N^{1-1/r}),
so temporary-space bound is asymptotically optimal.
```

### 时间优越性验证

记两组时间：

```md
Access:
- O(1) worst-case via Section 6.3

Grow/Shrink:
- O(r) amortized via credit argument
```

再连接下界：

```md
Theorem 9.1:
For standard implementations with storage extra O(rN^{1/r}),
grow amortized cost is Omega(r).

Construction achieves O(r).
Therefore grow/shrink amortized time is optimal up to constants
within standard implementations.
```
