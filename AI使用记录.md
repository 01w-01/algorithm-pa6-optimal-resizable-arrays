> 任务：帮助理解 §8 Growth Game 的递推公式

## 有效 Prompt

```
§8 的递推公式 C_k(a_1,...,a_k) = C_k(0,...,0,a_k) + C_{k-1}(a_1,...,a_{k-1})
为什么可以把代价拆成"最后一块"和"前面 k-1 块"两部分？
给出直观解释，并说明这个拆分与动态规划中"枚举最后一步"的对应关系。
```

## 回答要点

$A_k$ 最后一次被改变是在某次类型 (c) 合并中——那次合并把系统带到 $(0,\dots,0,a_k)$ 状态，此后 $A_k$**不再变化**。因此从全零到达 $\mathbf{a}$ 的整个过程可以切成两段：

1. 先到达 $(0,\dots,0,a_k)$——代价 $C_k(0,\dots,0,a_k)$
2. 在"$A_k$ 冻结"的前提下，独立操作前 $k-1$ 个子数组到达 $(a_1,\dots,a_{k-1})$——代价 $C_{k-1}(a_1,\dots,a_{k-1})$

两段之间**无耦合**（$A_k$ 已冻结，后续操作不触及它），所以总代价就是两项直接相加。这正是动态规划里"枚举最后一步"的标准用法：找到状态中一旦确定就不再变化的部分，把问题拆成独立子问题。

## 提炼

| 要素 | 说明 |
|------|------|
| 锁定不再变化的部分 | $A_k$ 冻结后，问题规模从 $k$ 降到 $k-1$，形成自然递推 |
| 类比动态规划 | 这和 DP 中"枚举最后一个决策"完全同构——递推公式 = 状态转移方程 |
| 论文中递推的作用 | 反复展开递推得到独立求和式，最终通向二项式系数的闭式解 |
